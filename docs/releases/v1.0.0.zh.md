# Agentic Method Workspace v1.0.0

首个公开版本，完成从本地产品工作区到独立方法仓库的收口。

## 本版包含

- 独立的 `agentic-method-kit`
- 可复制的 `agentic-repo-shell`
- 可选的 `sample-overlays/pantheon`
- 根级 Harness Engineering 契约文档
- bootstrap、验证、迁移与发布说明

## 适合谁使用

- 希望把 OpenSpec、task packet、evidence、review 组织成可机器校验闭环的团队
- 希望同时支持 Codex、Claude Code、Cursor、GitHub Copilot、OpenHands 与人工协作的团队
- 希望把“方法层”和“业务层”拆开治理的团队

## 核心能力

- 用 `agentic-method-kit` 维护方法事实源、模板、schema 与版本
- 用 `agentic-repo-shell` 复制仓库本地运行壳层
- 用 `docs/harness/*` 明确 Harness Engineering 契约
- 用 `scripts/harness/*` 做 adoption、health、task packet、evidence、review 等检查
- 用 `sample-overlays/pantheon` 展示项目专属继承治理 overlay 如何打包

## 推荐使用方式

普通新仓库复制：

1. `agentic-method-kit/`
2. `agentic-repo-shell/`
3. 如有类似继承治理需求，再参考 `sample-overlays/pantheon/` 改造项目自有 overlay

不要默认整仓复制当前 repo。

## 环境要求

- `git`
- `node` 20+
- `PowerShell`

Skills 不是必需依赖；skills 是执行加速器，仓库中的契约、模板与检查脚本才是事实源。
