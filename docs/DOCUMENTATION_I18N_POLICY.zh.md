# 文档双语维护策略

English version: [DOCUMENTATION_I18N_POLICY.md](./DOCUMENTATION_I18N_POLICY.md)

## 目标

这个仓库的文档要同时满足两类读者：

- 中文开发者可以直接理解和执行
- 国际协作方可以继续使用英文文档、稳定链接和发布面

## 结论

默认采用“中英分拆、互相链接”的策略。

推荐命名：

- 英文主文档：`README.md`
- 中文副本：`README.zh.md`

legacy 例外：

- 如果某份历史文档已经以中文正文稳定存在于 `*.md` 路径，并且已有外部引用，为避免破坏链接，可保留该中文主文档
- 这类文档新增英文副本时，使用 `*.en.md`

不推荐默认采用“同一文档上中文下英文”的形式，除非该文档极短，而且只作为公告或说明存在。

## 适用规则

### 1. 入口文档

入口文档必须优先提供中文版本。

典型范围：

- 根 `README`
- 目录级 `README`
- 安装、升级、迁移、发布、操作手册

### 2. 契约与规范

如果契约主要服务中文团队，可以先中文优先，再补英文副本。

如果契约已经被外部工具、外部仓库或公开文档引用：

- 保留现有英文路径不变
- 新增 `*.zh.md`
- 在两份文档顶部互相跳转

### 3. 模板与示例

模板文件优先保留英文结构，以减少机器消费和外部协作歧义。

中文支持建议通过以下方式补充：

- 中文导读
- 字段解释
- 单独的使用说明文档

### 4. Prompt、Adapter、Skill 文档

这类文档如果主要给 agent 或工具消费，不要求优先全文双语。

更适合：

- 保留英文原文
- 增加中文 README 或中文说明索引

## 维护规则

1. 英文原路径尽量不改，避免破坏已有链接
2. 中文文档统一使用 `.zh.md`
2.1 legacy 中文主文档可继续保留原始 `.md`，其英文副本使用 `.en.md`
3. 每份中英文文档顶部都加语言跳转
4. 同步更新时，先更新事实源，再更新翻译副本
5. 如两种语言内容暂时不完全一致，需在文档顶部标注以哪份为准

## 推荐模板

英文文档顶部：

```md
Chinese version: [README.zh.md](./README.zh.md)
```

中文文档顶部：

```md
English version: [README.md](./README.md)
```

## 分批推进建议

第一批：

- 根入口
- 方法入口
- 安装 / 升级 / 迁移 / 发布文档

第二批：

- overlay 文档
- repo shell 文档
- skills 索引

第三批：

- 模板说明
- adapter / prompt 导读
- 对外发布需要公开维护的英文契约副本
